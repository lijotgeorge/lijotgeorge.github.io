Rock Paper Scissors

2021-09-14

In the previous post I mentioned how I had skipped one lesson in the course before jumping into the Etch-a-Sketch project. Well it turns out I missed on a lot more than just learning about DOM maniupulation. There was a project to make a UI version of a simple rock-paper-scissors game taught earlier in the JavaScript course as well.

The earlier version was mainly to teach and get people familiar with the browser console and everything was printed there. It wasn’t all that hard for me then but making the UI had me pulling my hair for a good minute.

First, the demo of the project is here and the opening page is below.

It’s not quite obvious from the image but I wanted to go for a “Street-Fighter”-esque look for the game. Don’t think I quite succeeded with that but whatever. In implementation the task was to compare the choice made by the user with that of the “computer” which was just a random pick of the three choices. In the non-UI version the player choice was done using a prompt at the beginning of every round and the program would compare the two and decide on the winner based on the appropriate conditions.

For the UI, I had to overhaul basically everything. The only thing that remained was the function that decided the winner given the two choices. In this project as well there was a particular JS concept that was causing me a lot of trouble that I had to rely once again to the Discord users for help. The main thing I took away from the exercise was learning how to better use eventListeners. I have a slightly better understanding of the concept and hopefully that will help me out in my next projects. I know for sure if I had done this before the Etch-a-Sketch challenge I would not have had that hard of a time. Ah well, lessons learned I suppose.
